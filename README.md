# 🚀 Язык программирования Gopi (Gopi Lang)

**Gopi** — это функциональный, полный по Тьюрингу язык программирования со строгой статической областью видимости и ленивыми вычислениями (Lazy Evaluation).

Проект разработан на **F#** в рамках университетского курса по функциональному программированию. При создании интерпретатора намеренно **не использовались** генераторы парсеров (FsLex/FsYacc) — весь лексер и рекурсивный спуск (Top-Down Parser) написаны с нуля.

## 🏆 Чек-лист реализованных требований
- [x] Именованные переменные (`let`)
- [x] Рекурсия (через `func`)
- [x] Ленивое вычисление (Lazy Evaluation через Thunk)
- [x] Функции первого класса (именованные и анонимные)
- [x] Замыкания (Closures) и частичное применение (Currying)
- [x] Списки / Последовательности (`[...]`, `+`)
- [x] Библиотечные функции: списки (`map`, `fold`, `filter`, `head`, `tail`)
- [x] Библиотечные функции: ввод-вывод (`print`, `readFile`)

## 🚀 Быстрый старт

Убедитесь, что у вас установлен .NET SDK.

**Интерактивный режим (REPL):**
Запустите интерпретатор без аргументов, чтобы писать код прямо в консоли с сохранением контекста.
```bash
dotnet fsi gopi.fsx
```

**Запуск скриптов:**
Для выполнения готового файла `.gopi`:
```bash
dotnet fsi gopi.fsx docs/examples/test.gopi
```

## 🛠 Поддержка IDE (VS Code)
В репозитории есть скрипт `install_ext.fsx`. Запустите его (`dotnet fsi install_ext.fsx`), и он автоматически установит TextMate-грамматику для VS Code. Вы получите красивую подсветку синтаксиса для файлов `.gopi`!

## 👥 Команда разработки
Проект выполнен командой из 3 человек:
1. **[Имя 1]** — Разработка ядра (Лексер, Парсер, AST).
2. **[Имя 2]** — Среда выполнения (Evaluator, Ленивые вычисления, Замыкания).
3. **[Имя 3]** — Стандартная библиотека, интеграция с IDE, примеры программ на языке и документация.
