# Язык программирования Gopi

**Gopi** — это интерпретируемый функциональный язык программирования, разработанный на F# в рамках учебного проекта. Язык является полным по Тьюрингу и обладает строгой статической областью видимости.

Синтаксис языка намеренно сделан минималистичным (отсутствуют ключевые слова для объявления переменных, как в Python), однако вычислительная модель базируется на строгих функциональных концепциях: ленивых вычислениях, замыканиях и функциях первого класса. 

Интерпретатор написан с нуля без применения автоматических генераторов парсеров.

## 🚀 Режимы работы интерпретатора

Интерпретатор поддерживает два режима работы. Для запуска необходим установленный **.NET SDK**.

### 1. Интерактивный режим (REPL)
Если запустить интерпретатор без указания файла, откроется интерактивная среда (Read-Eval-Print Loop).
```bash
dotnet fsi gopi.fsx
```
**Подробности реализации REPL:**
* **Сохранение контекста (State Preservation):** В отличие от простых калькуляторов, REPL в Gopi сохраняет вычислительное окружение (`Env`) между запросами. Объявленные на одной строке переменные и функции доступны на следующих.
* **Автоматический вывод:** В интерактивном режиме не обязательно использовать функцию `print()`. REPL автоматически перехватывает результат вычисления выражения (число, строку, логическое значение или массив) и форматированно выводит его на экран.
* **Изоляция ошибок:** Если введенное выражение содержит синтаксическую ошибку или вызывает сбой выполнения (например, деление на ноль), REPL перехватывает исключение и выводит сообщение, сохраняя при этом все ранее определенные переменные.
* **Завершение работы:** Для выхода из консоли достаточно ввести команду `exit` или `quit`. Консоль завершит работу мгновенно и без лишних сообщений.

**Пример сессии:**
```text
Добро пожаловать в Gopi REPL!
Введите код или 'exit' для выхода.
gopi> a = 10
gopi> b = 5
gopi> a + b
15
gopi> arr = [1, 2, 3]
gopi> arr + [4, 5]
[1, 2, 3, 4, 5]
gopi> exit
```

### 2. Файловый режим (Запуск скриптов)
Для выполнения готового исходного кода передайте путь к файлу с расширением `.gopi` в качестве аргумента:
```bash
dotnet fsi gopi.fsx example.gopi
```
В файловом режиме автоматический вывод результатов в терминал отключен. Для отображения данных необходимо явно вызывать встроенную функцию `print()`.

## 👥 Команда разработки
Проект выполнен командой из 3 человек:
1. **[Имя 1]** — Разработка ядра (Лексический анализатор, Парсер, AST).
2. **[Имя 2]** — Среда выполнения (Evaluator, реализация ленивых вычислений через Thunk, REPL).
3. **[Имя 3]** — Стандартная библиотека, интеграция функций высшего порядка, написание алгоритмов (Quicksort) и документация.
