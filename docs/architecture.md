# Архитектура интерпретатора

## 1. Абстрактное синтаксическое дерево (AST)
Синтаксис программы отображается в алгебраический тип данных `Expr`.
```fsharp
type Expr =
    | Num of int
    | Var of string
    | Assign of string * Expr
    | Lambda of string list * Expr
    | Func of string * string list * Expr
    | Call of Expr * Expr list
    | If of Expr * Expr * Expr
    // и т.д.
```

## 2. Лексер и Парсер (Top-Down Parser)
* **Лексер (`tok`)**: выполняет посимвольный проход по строке, разбивая её на токены и отсекая комментарии (`//`).
* **Парсер (`pe`, `pt`, `pb`)**: реализует алгоритм рекурсивного спуска. Функции взаимно рекурсивно вызывают друг друга для корректной обработки приоритетов математических операций и вложенных блоков `{ ... }`.

## 3. Среда выполнения (Evaluator)
Главная функция `ev` принимает окружение (`Env = Map<string, Value>`) и AST-выражение, возвращая вычисленное значение и, при необходимости, обновленное окружение.

**Реализация ленивости (Thunk):**
Для отложенных вычислений используется тип:
```fsharp
| VThunk of Expr * Env ref * Value option ref
```
При операции присваивания интерпретатор не вызывает `ev`, а создает `VThunk`, запоминая AST-узел и ссылку на текущее окружение. Когда переменная реально используется, вызывается функция `force`. Она вычисляет значение и кэширует его в `Value option ref` (мемоизация), предотвращая повторные вычисления в будущем.

## 4. Интерактивный цикл (REPL)
Функция `repl` реализована через хвостовую рекурсию. Она принимает текущее состояние `Env`. После ввода пользователем строки кода, она парсится, вычисляется через `ev`, а полученное новое окружение `newEnv` передается в следующий рекурсивный вызов `repl newEnv`. Это позволяет накапливать переменные и функции в памяти в ходе одной сессии.
