# ⚙️ Архитектура интерпретатора

Интерпретатор Gopi написан на чистом **F#** и состоит из 4 основных логических блоков. Мы отказались от использования FsLex/FsYacc в пользу прозрачного и контролируемого ручного парсинга.

## 1. Абстрактное Синтаксическое Дерево (AST)
Определено через алгебраические типы данных (Discriminated Unions).
```fsharp
type Expr =
    | Num of int
    | Var of string
    | Assign of string * Expr
    | Lambda of string list * Expr
    | Call of Expr * Expr list
    | If of Expr * Expr * Expr
    // ...
```

## 2. Токенайзер (Лексер)
Функция `tok` посимвольно читает исходную строку и разбивает её на значимые токены, игнорируя пробелы и комментарии (`//`). Обрабатывает сложные операторы вроде `==`.

## 3. Парсер (Top-Down Recursive Descent)
Функции `pe`, `pt` и `pb` реализуют алгоритм рекурсивного спуска. 
Парсер автоматически обрабатывает приоритет операций (через вложенность вызовов) и строит древовидную структуру `Expr`.

## 4. Вычислитель (Evaluator)
Функция `ev` принимает окружение (`Env = Map<string, Value>`) и выражение (`Expr`).
Для реализации ленивых вычислений используется тип `VThunk`:
```fsharp
| VThunk of Expr * Env ref * Value option ref
```
Когда переменная связывается через `let`, интерпретатор не вычисляет её, а создает Thunk. При первом чтении переменной срабатывает функция `force`, которая вычисляет выражение и кэширует результат (мемоизация) в `Value option ref`.
